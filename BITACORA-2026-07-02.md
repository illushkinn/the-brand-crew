## 🔍 PROBLEMAS IDENTIFICADOS (No Resueltos)

### Crítico - Breakpoints Visuales
**Ubicaciones**:
- Arriba de "Three steps. No fluff" (Como Funciona section)
- Arriba de "A steal" (Pricing section)

**Causa Raíz**:
- Cada sección tiene su propio contenedor de shapes (`.hero-shapes`, `.pricing-shapes`, `.faq-shapes`, `.cta-shapes`)
- Esto crea cortes visuales entre secciones
- Gradiente de fondo se ve cortado/interrumpido

**Impacto Visual**:
- Breakpoint/línea horizontal visible
- Fondo no es continuo
- Rompe la ilusión de "vidrio glassmórfico unificado"

**Solución Propuesta** (Documentada en `ANALISIS-VISUAL.md`):
1. Crear UN SOLO contenedor `.global-shapes` 
2. Posicionar 6-8 shapes estratégicamente para cubrir toda la página
3. Eliminar contenedores individuales por sección
4. Shapes con `position: fixed` para efecto continuo

### Performance - Demasiadas Shapes
**Actual**:
- Hero: 9 shapes (desktop) / 4 (mobile) → DESACTIVADAS en mobile
- Pricing: 3 shapes
- FAQ: 3 shapes
- CTA: 3 shapes
- Total: ~18 shapes animándose

**Impacto**:
- GPU usage alto
- Animaciones costosas
- Mobile especialmente afectado (por eso se desactivaron)

**Solución Ideal**:
- 6-8 shapes globales (no por sección)
- Fixed positioning para continuidad
- Blur más grande pero menos shapes

## 🎨 PLAN DE GLASSMORPHISM UNIFICADO

### Concepto
"Todo el contenido (cards bento) levitando sobre un enorme vidrio glassmórfico continuo con manchas de colores (esferas) que se mueven aleatoriamente."

### Implementación Propuesta

#### Paso 1: Crear Contenedor Global
```html
<!-- Agregar antes de </body> -->
<div class="global-shapes" aria-hidden="true">
  <div class="shape shape-1"></div>
  <div class="shape shape-2"></div>
  <div class="shape shape-3"></div>
  <div class="shape shape-4"></div>
  <div class="shape shape-5"></div>
  <div class="shape shape-6"></div>
  <!-- Desktop only: -->
  <div class="shape shape-7"></div>
  <div class="shape shape-8"></div>
</div>
```

#### Paso 2: CSS Base
```css
.global-shapes {
  position: fixed;
  inset: 0;
  z-index: -1;
  pointer-events: none;
  overflow: hidden;
  will-change: transform;
  transform: translateZ(0);
}

.shape {
  position: absolute;
  border-radius: 50%;
  filter: blur(80px);
  opacity: 0.15;
  will-change: transform;
}
```

#### Paso 3: Posicionamiento Estratégico
```css
/* Distribución vertical para cubrir toda la página */
.shape-1 {
  width: 600px; height: 600px;
  background: linear-gradient(135deg, var(--terracota), #dd614c);
  top: 10%; left: 15%;
  animation: float1 30s infinite;
}

.shape-2 {
  width: 500px; height: 500px;
  background: linear-gradient(225deg, var(--verde), #8cd44a);
  top: 40%; right: 10%;
  animation: float2 35s infinite;
}

.shape-3 {
  width: 450px; height: 450px;
  background: linear-gradient(180deg, var(--terracota), #c47a68);
  bottom: 20%; left: 20%;
  animation: float3 40s infinite;
}

/* Mobile: ocultar shapes 5-8 */
@media (max-width: 767px) {
  .shape-5, .shape-6, .shape-7, .shape-8 {
    display: none;
  }
}
```

#### Paso 4: Eliminar Shapes Antiguas
```css
/* Ocultar contenedores antiguos */
.hero-shapes,
.pricing-shapes,
.faq-shapes,
.cta-shapes {
  display: none !important;
}
```

1. **PowerShell escape characters**: Dejó `\`n` en el código → Solucionado con búsqueda manual
2. **Performance con 18+ shapes**: Causaba drops a 20-25 FPS en mobile → Desactivadas en mobile
3. **Vercel 404**: Al final de sesión, posible issue con config → Pendiente verificar
4. **WSL file system**: Mencionado en contexto, aparentemente resuelto

### Mejores Prácticas Aplicadas
- Commits atómicos y descriptivos
- Testing incremental con localhost
- Documentación exhaustiva en .md files
- Performance-first approach (mobile)
- Accessibility (ARIA labels, skip links, semantic HTML)

### Decisiones de Diseño
- Mobile-first: Desactivar shapes antes que degradar experience
- Glassmorphism: Priorizar continuidad visual sobre cantidad de shapes
- Micro-interactions: Sutiles (2px translateY) para no afectar performance
- Coherencia: Section labels en todas las secciones principales
